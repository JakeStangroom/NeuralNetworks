6  Networking

Neural networks are highly flexible. That is their strength and their risk.

A network with many hidden units can often fit the training data very well, but throughout the course, we have already seen how this can be an issue. The network that fits the training data best is not necessarily the network that generalises best. We want to create a model which performs well on out-of-sample data, not a model that learns the noise present in the training data.

This chapter represents a collection smaller ideas which enhance the larger concepts already covered.

6.1 Input Preprocessing

Gradient Descent, hence the parameter optimisation process, is sensitive to the scale of the predictors. If one predictor is measured in thousands and another in decimals, the corresponding weights can learn at very different rates. We saw this in our process of fitting SVMs.

For this reason, neural-network predictors are usually standardised:

\[x_{ij}^{std} = \frac{x_{ij} - \bar{x}_j}{s_j}.\]

That is, each input variable is centred at zero and scaled to have standard deviation one.

NoteA small warning about preprocessing

The standardisation parameters must be estimated from the training data only. The same training means and standard deviations are then applied to the validation and test data.

If the validation or test data influence the scaling parameters, information leaks from held-out data into the training process.

6.2 Hyperparameter Tuning

In previous sections, we have had to make model choices which directly affect the bias-variance tradeoff. These parameters intend to offer control over model overfitting, and we tuned them with some form of validation analysis. Neural networks are no different.

6.2.1 Model Size

For a fixed input and output dimension, the number of hidden nodes and hidden layers controls the model size. More hidden nodes and layers requires more connections which are characterised by the weights which manage the linear transformations. Naively adding extra nodes and layers can quickly escalate the number of model weight parameters required to optimise.

More weights can reduce bias because the model can express more complicated functions; whilst more weights can also increase variance because the model becomes flexible enough to learn the training noise, rather than the underlying relationships. This is our standard bias-variance tradeoff that we have been grappling with throughout the course.

There is no universal number of hidden nodes or layers that works best, it is very problem specific. In practice, we choose model size using validation performance.

6.2.2 Random Initialisation

Neural network training generally starts from random weight values. When we consider the loss function as a function of a large amount of parameters, each of which may have a complex connection to the loss, there is no reason that the loss function is convex with respect to each of these parameters. In reality, the loss function is generally non-convex and have many local minima, and other difficult regions.

For this reason, two identical networks can give different fitted models if they start from different initial weights. The path of optimisation which they take through the loss function defines the local minima that they encounter; some of which may to be too large to escape.

For this reason, neural network fitting can repeat the training process for several random initialisations; however, this can be very time consuming. Furthermore, it is clear that setting a seed is essential to achieve reproducibility.

6.2.3 Regularisation

In order to manage the bias-variance tradeoff in other model architectures, we have considered adding extra terms to the objective function. This term penalises a characteristic of a model which provides too much flexibility in the model, promoting overfitting.

In other sections of the course, we have seen the use of \(L1\) and \(L2\) regularisation. The same notion exists for neural networks. Large weights are sensitive to inputs, and can increase the flexibility of the model, hence these regularisation penalties try to mitigate large weights.

For squared-error regression, an \(L2\) regularised objective can be written as

\[ \mathcal{L}_{\lambda}(\mathbf{w}) = \frac{1}{2n} \sum_{i=1}^n \left(y_i - \hat{y}_i\right)^2 + \lambda \sum_j w_j^2. \]

In neural networks, an \(L2\) regularisation is also often called weight decay, but this is simply a topic specific name for something we have already encountered. The penalty encourages the network to use smaller weights which often produce smoother fitted functions and reduce overfitting.

Alternatively, we could use \(L1\) regularisation instead. Whilst it does not have a special neural network name, it is still a relevant form of regularisation. Again, it tries to nudge the network to optimise with smaller weights, but it just does so in a slightly different way.

The parameter \(\lambda\) controls the strength of the penalty:

  • small \(\lambda\): weak penalty, more flexible network
  • large \(\lambda\): strong penalty, more constrained network

6.3 Early Stopping

Another way to control model complexity is early stopping.

During training, the training loss usually decreases as the network continues to update its weights. The validation loss often decreases at first, but may start increasing once the model begins to overfit. Early stopping stops training the model near the point where validation performance is best.

The idea is simple:

  1. split the data into training and validation sets
  2. train the network iteratively
  3. monitor validation loss
  4. stop once validation performance stops improving

The early stopping process treats the number of training iterations as a regularisation choice; attempting to limit the training iterations to those which are not overfitting.

6.4 Validation Workflow

A useful neural network workflow is:

  1. Choose a validation procedure such as cross validation.
  2. Standardise predictors using training data only.
  3. Choose a small set of candidate hyperparameter values.
  4. Train each candidate model with validation procedure.
  5. Compare with validation procedure
  6. Select and refit the final model on all training data.

This workflow is not unique to neural networks, but it is especially important for them because they can be sensitive to architecture, learning rate, preprocessing, and random initialisation.
