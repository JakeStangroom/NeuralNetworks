4  Training By Gradient Descent

When we talk about training a neural network, this means we are tackling the optimisation problem of choosing the weights and biases. For a neural network, this optimisation controls the feature transformation; hence, we are training the neural network to learn the important features required for the modelling question. In some of the previous models, we have been able to solve the model parameters analytically. With the neural network architecture, this becomes unrealistic very quickly. Even a modest network can have many weights; where modern network architectures can have millions or billions of these parameters to optimise. In fact, ChatGPT 5 is estimated to have 635 billion parameters.

Clearly, we need an iterative approach to solving such a complex optimisation. The central idea of this iterative approach is:

choose a loss function, then move the parameters in a direction that makes the loss smaller.

4.1 Loss Functions

We already introduced the idea of a loss function in Chapter One, but let us introduce it more formally here.

4.1.1 What is a Loss Function?

Let \(y\) and \(\hat{y}\) have their standard interpretations of the true response and predicted response respectively. A loss function is a function \(\mathcal{L}\) that takes both \(y\) and \(\hat{y}\) as inputs, and returns a real number which captures how poorly the model predicted the response. Therefore, it is common that

  1. \(\mathcal{L}(y,\; \hat{y}) \geq 0,\)
  2. \(\mathcal{L}(y,\; \hat{y}) = 0,\) when \(y = \hat{y}\).

A large loss value shows that the model performed poorly, and should be penalised; yet, a small loss value shows that the models performance was very good. It is common to have a smooth loss function for mathematical convenience; but, it is not necessary. The amount of penalty to apply to prediction errors can be rather subjective; however, there are common conventions used which make intuitive sense and benefit from mathematical ease down the line.

4.1.2 Common Choices

For regression, a common choice is mean squared error:

\[\mathcal{L}(y, \;\hat{y}) = \frac{1}{n} \sum_{i=1}^n \left(y_i - \hat{y}_i\right)^2.\] The MSE makes the assumption that we incur a loss whenever the prediction does not match the output perfectly. Furthermore, we are indifferent to whether the prediction is above or below the true value, the penalty is only based on the distance from the true value. The penalty then becomes much larger the further distance from the true value.

For binary classification, a common choice is cross-entropy:

\[\mathcal{L}(y, \; \hat{p}) = -\frac{1}{n} \sum_{i=1}^n \left[ y_i\log(\hat{p}_i) + (1-y_i)\log(1-\hat{p}_i) \right].\]

Here, the loss represents how far away our predicted class probability is from the observed class. The loss is minimised when the prediction probability is correctly certain about the class of the observations.

Therefore, the loss is the quantity the training algorithm tries to minimise since the model wants to perform as well as it can, at least on the training data.

These choices connect back to the single-neuron examples. A linear output unit trained with squared error is linear regression. A sigmoid output unit trained with cross-entropy is logistic regression. A threshold perceptron does not fit either of these smooth losses in its basic form; it uses a direct mistake-correction rule instead.

It is important to note that minimising the loss function is implicit in its expression of the optimisation parameters. Remember that whilst \(y\) is the observed true value, \(\hat{y}\) is the predicted model output which relies on the inner model parameters; in this case, the set of weights and biases. As a result, it is common to express the loss as a function of these parameters explicitly with a function, \(J(\mathbf{w}, b)\), for example. This function, essentially an objective function, is the loss associated with \(\mathbf{w}\) and \(b\) as the model parameters. Such an expression makes the optimisation problem clearer, but does not change the meaning of anything.

4.1.3 The Simplest Example

Let us begin with the smallest possible regression example. Suppose the true relationship is

\[y = 0 + 2x.\]

If we ignore random noise, for now, then a collection of data may be:

0 0
1 2

Since we have a basic regression problem, we will choose a single neuron with a linear activation function and loss function equal to half the MSE. Since we will only have a single predictor, we want to find the parameters \(w_1\) and \(b\) which minimise this loss. (Technically, denoting the weight with a subscript is overkill here; however, it is beneficial for when we extend the notation to more predictors later on.)

For simplicity, suppose that we fix \(b = 0\) and try to optimise for just \(w_1\) in the linear model

\[\hat{y} = w_1x + b.\] We can begin by trying to get an understanding of the objective function by evaluating the loss for a collection of values for \(w_1\).

4.2 The Gradient Descent Update

Recall the approach for optimising the classical perceptron. It was made up of two steps, a forward step and an update step. What we have done above is essentially the forward step. We started with some initial parameters and evaluated the model with those parameters. Now, we need to establish the update step.

For the classical perceptron, the update was taken when the model misclassified the observation, a mistake-driven update which perturbed the boundary to make it more likely to be correctly. To extend the notion of slightly moving the boundary in the right direction after a mistake, we can call on the natural optimisation solution; gradients.

Based on the prediction error of the model, move the parameters in a better direction with the gradient of the objective function.

NoteWhat is a gradient?

We all know what a gradient is; however, there is an interpretation of a gradient which is less commonly thought of. This may be trivial to some, but to explicitly state it may help others.

The gradient tells us the direction which increases the function.

For example, let \(f(x)=\frac{1}{2}x^2\) and consider \(\frac{\partial f}{\partial x} = x\). When \(x<0\), the gradient is negative implying that if we move \(x\) in the negative direction, \(f(x)\) will increase. When \(x>0\), the gradient is positive implying that if we move \(x\) in the positive direction, \(f(x)\) will increase.

If we want to move towards the minimum of the function \(f\), then we must move in the opposite direction to the gradient.

Therefore, we can define the Gradient Descent update equation for each weight using the slope of the objective function such that

\[w_j \leftarrow w_j - \eta\; \frac{\partial }{\partial w_j}J(\mathbf{w}, b),\] and

\[b \leftarrow b - \eta\; \frac{\partial }{\partial b}J(\mathbf{w}, b).\]

Again, \(\eta\) is the learning rate. The partial derivative tells us the slope of the objective function with respect to that parameter. This is interpreted as the direction of steepest local increase in the loss; hence, the minus sign means we move in the opposite direction, toward lower loss.

Choosing differentiable functions allows us to leverage the partial derivatives.

The learning rate determines the size of the step we take toward the minimum:

  • if \(\eta\) is too small, learning can be painfully slow; or even worse, it can get stuck in local minima
  • if \(\eta\) is too large, the algorithm can overshoot and fail to converge

In these examples, it seems trivial to find the optimal solution with the algorithm. The partial derivative is smooth and its just a matter of time. Things can escalate very fast

4.3 More Than One Parameter

With two parameters, the loss is a surface rather than a curve and we find ourselves visualising the minimisation with a contour plot. Note that this extension does not arrive from a new predictor. It is inherent in the simplest case of a single predictor since we need to find both the weight and bias parameters. Thankfully, the same idea still applies, Gradient Descent just moves across this surface towards an area of lower loss, aiming to minimise the objective function.

Therefore, in any neural network that is not this toy example, the surface exists in a (much) high-dimensional parameter space. We cannot visualise it, but Gradient Descent algorithm still uses local slope information to decide how to move each parameter.

This is the general intuition of the Gradient Descent algorithm; however, there are still some outstanding details. For example, what level of granularity do we consider these observations?

4.4 Types of Gradient Descent

In our classical perceptron case, we chose to evaluate the error of the model on a single observation. For the Gradient Descent problem, we have multiple choices on how exactly we choose the error. We could continue with a single observation for the error; hence, updating the equations based on each single observation. Just because we used this approach for the perceptron does not mean that it is the only logical approach though. In fact, there are several common ways to compute the gradient and update step.

  1. Stochastic gradient descent updates the parameters after one training observation. This is faster per update and can work well at scale, but the updates are noisy.

  2. Batch gradient descent computes the gradient using all training observations before updating the parameters. This is stable, but can be slow for large datasets.

  3. Mini-batch gradient descent represents a compromise. It computes the gradient on a small batch of observations, such as 32, 64, or 128 observations, then updates the parameters.

The modern default is usually mini-batch learning.

4.5 Takeaway

Training a neural network requires three separate choices:

  • the loss function measures the penalty for poor prediction quality
  • the type gradient descent algorithm defines how the parameters are updated and improved upon
  • the learning rate controls the size of those updates

Gradient Descent represents the basic optimisation idea. We still have a major problem; how do we actually calculate these gradients in an efficient way?
